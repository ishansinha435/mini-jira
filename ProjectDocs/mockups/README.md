## Using These Mockups

### For Implementation
1. Review `MOCKUP_SPECIFICATIONS.md` for detailed specs
2. Follow principles from projectContext.md and Cursor rules
3. Use shadcn/ui components (no custom replacements)
4. Focus on visual hierarchy and UX states, not pixel-perfect sizing

### Key Principles
- Modern, clean, "startup-grade" UI
- Consistent spacing using Tailwind scale
- shadcn/ui primitives for all components
- Loading, empty, and error states for all views
- Mobile-first responsive design
